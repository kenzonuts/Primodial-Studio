# Future Roadmap

Prioritized for product growth **after** stable homepage launch. No redesign of existing sections implied unless called out.

## Phase 1 — Harden (0–30 days)

- Live Lighthouse ≥95 remediation loop
- Rate limiting on public APIs
- Production error monitoring (Sentry)
- Real portfolio imagery & case blurbs
- Favicon / brand asset pack polish
- Cross-browser sign-off matrix complete

## Phase 2 — Depth pages (30–90 days)

- Dedicated **About** page
- **Services** index + `/services/[slug]`
- **Work** index + case studies `/work/[slug]`
- Re-enable dynamic sitemap entries
- Structured data for CreativeWork / Service

## Phase 3 — Content engine (90–180 days)

- **Blog** expansion (authors, categories, RSS)
- Editorial workflow in Payload (roles already modeled)
- Case study templates with metrics blocks
- Internal linking strategy from blog → services → contact

## Phase 4 — International & theme

- **Multi-language** (locale routing + content fields)
- Refined **dark / light** theme parity QA
- Localized SEO metadata + hreflang

## Phase 5 — Platform

- **Authentication** (studio clients)
- **Client Portal** (files, milestones, invoices summary)
- **Admin Dashboard** beyond Payload (ops metrics)
- **Project tracking** for engagements
- **Live chat** / qualified lead routing
- **AI Assistant** for site Q&A (RAG over public content only)

## Explicit non-goals (near term)

- Redesigning the homepage narrative without product brief
- Changing the design-system token language casually
- Shipping thin pages solely for keyword targeting

## Suggestion backlog → intake

New ideas enter as issues with: problem, audience, success metric, a11y impact, SEO impact, eng estimate.
